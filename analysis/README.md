# Analysis Pipeline

R scripts to prepare RNA-seq count data and run differential expression analysis. The output CSV can be uploaded directly to the Transcriptomics Explorer dashboard.

---

## Workflow

```
fetch_geo.R  (Option A: airway built-in dataset, OR Option B: any GEO accession)
    ‚Üì
counts.csv + metadata.csv
    ‚Üì
deseq2_analysis.R  (or edger_analysis.R)
    ‚Üì
de_results.csv  (with gene symbols)
    ‚Üì
Upload to dashboard ‚Üí all 6 tabs update instantly
```

---

## Files in this folder

| File | Purpose |
|------|---------|
| `fetch_geo.R` | Get a count matrix ‚Äî either from the built-in airway dataset or from GEO |
| `deseq2_analysis.R` | Run DESeq2 ‚Üí outputs `de_results.csv` with gene symbols |
| `edger_analysis.R` | Alternative to DESeq2 (better for < 3 replicates per group) |
| `counts.csv` | Count matrix generated by fetch_geo.R (airway dataset) |
| `metadata.csv` | Sample info generated by fetch_geo.R |
| `de_results.csv` | Final output ‚Äî upload this to the dashboard |

---

## Requirements

**R ‚â• 4.1** is required. All packages install automatically on first run.

Packages installed automatically:
- `airway` ‚Äî built-in Bioconductor RNA-seq teaching dataset (Option A)
- `GEOquery` ‚Äî download datasets from GEO (Option B)
- `DESeq2` ‚Äî differential expression (recommended)
- `edgeR` + `limma` ‚Äî alternative DE method
- `org.Hs.eg.db` + `AnnotationDbi` ‚Äî gene symbol annotation (human)
- `ashr` ‚Äî fold change shrinkage
- `ggplot2`, `dplyr` ‚Äî plotting and data manipulation

For **mouse** data, also install: `BiocManager::install("org.Mm.eg.db")`

## Step 1 ‚Äî Get count matrix

```r
# In RStudio: open fetch_geo.R and click Run
# In Terminal:
Rscript fetch_geo.R
```

`fetch_geo.R` has two options at the top ‚Äî set `USE_AIRWAY`:

```r
USE_AIRWAY <- TRUE    # TRUE = airway built-in (no download, recommended for testing)
USE_AIRWAY <- FALSE   # FALSE = download from GEO using GEO_ID below
GEO_ID     <- "GSE96870"
```

**Option A ‚Äî airway (default, recommended for testing):**
- No internet download required
- 8 samples, ~16,000 genes after filtering
- Experiment: dexamethasone drug treatment vs untreated, human airway cells
- Produces the `de_results.csv` included in this folder

**Option B ‚Äî GEO download:**
- Downloads any GSE accession that provides a supplementary count matrix file
- NOTE: after downloading, open `metadata.csv` to find the correct column names for Step 2

---

## Step 2 ‚Äî Run differential expression

### Option A: DESeq2 (recommended, ‚â• 3 replicates per group)

```r
Rscript deseq2_analysis.R
```

### Option B: edgeR (works with fewer replicates)

```r
Rscript edger_analysis.R
```

**Edit the USER SETTINGS at the top of the script before running:**

For the **airway dataset** (default):
```r
SAMPLE_ID_COL <- "sample_id"
CONDITION_COL <- "dex"
GROUP_A       <- "untrt"    # reference / control
GROUP_B       <- "trt"      # treatment / case
GENE_ID_TYPE  <- "ENSEMBL"  # airway uses Ensembl IDs
ANNOTATION_DB <- org.Hs.eg.db  # human annotation
```

For a **GEO dataset**, first inspect `metadata.csv`:
```r
meta <- read.csv("metadata.csv")
colnames(meta)                       # find the condition column name
table(meta$characteristics_ch1)      # see exact group label text
```
Then set `SAMPLE_ID_COL`, `CONDITION_COL`, `GROUP_A`, `GROUP_B` accordingly.

**Gene annotation:**
The scripts automatically map Ensembl IDs (`ENSG00000...`) to gene symbols (`TP53`, `BRCA1`) using `org.Hs.eg.db`. Genes that cannot be mapped keep their Ensembl ID. For mouse data, change `ANNOTATION_DB <- org.Mm.eg.db`.

**Output files:**
| File | Description |
|------|-------------|
| `de_results.csv` | DESeq2/edgeR results with gene symbols ‚Üí upload to dashboard |
| `ma_plot.pdf` | MA plot QC check |
| `volcano_plot.pdf` | Volcano plot preview |

## Step 3 ‚Äî Upload to dashboard

1. Start the dashboard from the parent folder:
   ```bash
   python app.py
   ```
2. Open **http://localhost:7860** in your browser
3. Click **Upload** and select `de_results.csv`
4. All 6 tabs update automatically

## Column format of de_results.csv

| Column | Description |
|--------|-------------|
| `gene` | Gene symbol (e.g. TP53) ‚Äî or Ensembl ID if symbol not found |
| `ensembl_id` | Original Ensembl ID (e.g. ENSG00000141510) |
| `baseMean` | Average expression across all samples |
| `log2FoldChange` | Log2 ratio of GROUP_B / GROUP_A |
| `pvalue` | Raw p-value |
| `padj` | Adjusted p-value (Benjamini-Hochberg FDR) |
| `regulation` | "Up", "Down", or "NS" based on thresholds |

## Choosing between DESeq2 and edgeR

| | DESeq2 | edgeR |
|-|--------|-------|
| Best for | ‚â• 3 replicates per group | 2‚Äì3 replicates, smaller datasets |
| Normalisation | Median of ratios | TMM |
| Fold change | Shrinkage via ashr (more conservative) | Raw logFC |
| Gene annotation | Yes ‚Äî via org.Hs.eg.db | Yes ‚Äî via org.Hs.eg.db |
| Speed | Slightly slower | Slightly faster |

Both produce equivalent results on well-powered experiments. DESeq2 is the current community default.
